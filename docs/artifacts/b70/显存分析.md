# B70 显存分析

## 1. 静态结论

`workflow_memory_assessor.py` 对 B70 workflow 的静态评估结果：

- `projected_total_xpu_weight_gb = 225.98`
- 单卡 24 GiB 下，**绝不可能**把全部权重常驻 XPU
- 结论不是“少调一点就行”，而是：
  - UNet / CLIP 必须 CPU-biased
  - VAE 可以保留 XPU 候选
  - 真正要防的是 **Wan denoise 的激活/临时张量峰值**

静态建议：

| 组件 | 静态评估 |
| --- | --- |
| 低噪 UNet | XPU 投影约 37.44 GiB，必须 CPU-biased |
| 高噪 UNet | XPU 投影约 27.06 GiB，必须 CPU-biased |
| T5 文本编码器 | 单个约 10.59 GiB，优先 CPU |
| VAE | 约 0.24 GiB，可保留 XPU 候选 |

## 2. smoke 与 full-size 的几何差

本次为了隔离“几何规模”而不是“采样步数”，full-size probe 保持：

- steps 固定为 `8`
- 只恢复原始几何：
  - `54`: `1024 / 81`
  - `208`: `97` 帧

所以失败可以归因到 **几何规模本身**，而不是“步数太高”。

### 2.1 `54` 路径的几何放大

`54` smoke 使用 `512 / 17`，full-size probe 恢复到 `1024 / 81`。

近似放大量：

- 空间: `1024^2 / 512^2 = 4x`
- 时间: `81 / 17 ~= 4.76x`
- 组合放大: `~19.06x`

这解释了为什么同一路径 smoke 可过，而 full-size geometry 会在采样阶段直接崩掉。

### 2.2 `208` 路径的几何放大

`208` smoke 使用 `17` 帧，full-size probe 恢复到 `97` 帧。

仅时间维放大：

- `97 / 17 ~= 5.71x`

即使空间没有像 `54` 那样放大，时间维也足以把 Wan FFN / activation scratch 顶到 XPU 上限。

## 5. 为什么 `208` 在 20.97 GiB 仍然会 OOR

粗采样 telemetry 看到的是 **每 2 秒一次的外部观测值**，而不是瞬时 kernel / scratch 分配峰值。

因此：

1. `xpu-smi` 的 20.97 GiB 不是“绝对峰值”
2. 真正失败点可能来自：
   - FFN/GELU 前后的瞬时扩张张量
   - oneAPI / Level Zero runtime 的额外临时缓冲
   - allocator 碎片化
   - cleanup 前后的同步 / cast buffer 释放延迟

这和 `54` 的结论一致：**瓶颈不是权重常驻，而是 Wan denoise 激活峰值与临时分配。**

## 6. 根因归纳

### 不是这些问题

- 不是节点没加载
- 不是 API prompt schema 错
- 不是缺少 custom node patch
- 不是 CLIP / UNet device 设成 CPU 以后就自然解决
- 不是 generic `lowvram` 单独就能抹平 full-size 峰值

### 是这些问题

1. **full-size 几何规模下，Wan denoise 激活峰值超过单卡 B60 / 24 GiB 的有效 headroom**
2. `54` 与 `208` 都在 Wan 计算主体内部失败，说明这是结构性上限，不是某一条分支独有的小 bug
3. cleanup 后再触发 `DEVICE_LOST`，说明失败后恢复路径也不够稳，需要重启或至少重建上下文

## 7. 收敛后的策略建议

### 现阶段有效

- 保持 CPU-biased loader policy
- 单独做 smoke / 结构验证
- 把缺失素材明确标记成 smoke-only alias

### 若要追 full-size

优先级建议：

1. **多卡 / 分布式切分**
2. **Wan 内部 activation 级优化**
   - attention / FFN 分块
   - 时序块切分
   - 参考帧分批
3. **更激进的低精度替代**
4. **小分辨率生成 + 后处理**

不建议继续把时间花在：

- 泛化重试 `--lowvram`
- 单纯加 cleanup 节点
- 只做权重 offload 而不碰激活路径
