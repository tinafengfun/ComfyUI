## 4. full-size geometry probe 结果

### 4.1 分支 `54`

- 配置：原始 `1024 / 81`，steps=`8`
- 结果：失败
- 失败点：`KSamplerAdvanced -> model.apply_model`
- 错误：`UR_RESULT_ERROR_OUT_OF_RESOURCES`
- telemetry 峰值：`23323.75 MiB`

### 4.2 分支 `208`

- 配置：原始 `97` 帧，steps=`8`
- 结果：失败
- 失败点：Wan block `FFN -> GELU`
- 错误：
  - `UR_RESULT_ERROR_OUT_OF_RESOURCES`
  - cleanup 阶段 `UR_RESULT_ERROR_DEVICE_LOST`
- telemetry 峰值：`20972.01 MiB`

## 5. 失败归因

两条 full-size 路径都失败在 Wan denoise 内部，且 smoke 已经证明迁移链路本身可用，因此这次 B70 迁移的阻塞点应归类为：

> **单卡 B60 / 24 GiB 下，full-size geometry 的 Wan activation / scratch 资源上限**

不是：

- 节点没加载
- prompt schema 错
- 少补某个 custom node patch
- asset inventory 不完整

补充实验结论：

- 已额外尝试一次 **XPU `sub_quad` attention override** 来降低 full-size `54` 的激活峰值
- 结果仍然失败，且失败点提前到了 Wan block 入口的 modulation cast
- 因此目前不能把问题归因成“默认 PyTorch attention 路径选错了”
